## Required Functionality:

**Functionality 1:** 
    * Creates a directory.
    * **Purpose:** The core functionality being tested is the creation of a directory by `REDUCED_BINARY`.
    * 
**Functionality 2:** 
    * Sets permissions of a newly created directory using the `-m` flag with valid octal permission values (e.g., 400, 500, 644, 700).
    * **Purpose:**  This tests the ability of `REDUCED_BINARY` to correctly handle the `-m` flag for setting directory permissions. 
    * 
**Functionality 3:**
    * Produces an error message when provided with invalid input for the `-m` flag.
        * **Invalid Input 1:**  Missing permission value after `-m`.
        * **Invalid Input 2:**  Invalid octal permission value (e.g., 999).
    * **Purpose:** This tests the error handling of `REDUCED_BINARY` when given incorrect input related to the `-m` flag.
    * 
## Unrequired Functionality (to be removed):

1. **Handling of flags other than `-m`:** The test cases only focus on the `-m` flag, so any other flag functionality is unnecessary.
2. **Creating parent directories:** The test cases always attempt to create a directory "d1" directly, so the `-p` (parents) flag functionality is not being tested.
3. **Verbose output:** The test cases do not check for verbose output using the `-v` flag.
4. **Help and version information:** The test cases do not utilize the `--help` or `--version` flags.
5. **Specific error message content:** The test cases only check if an error message is produced for invalid `-m` usage, not the specific content of the message.
